# Data Dictionary — NCAA Next Man Up

This document provides column-level documentation for every CSV file in the `data/` directory. Files are organized by subfolder.

---

## `data/features/`

These files contain engineered features, percentile scores, archetype assignments, and dimension scores generated by the feature engineering and evaluation engine stages of the pipeline.

---

### `player_archetype_assignment.csv`
Archetype classification for each player, determined by scoring their normalized metric profile against all 7 archetype templates and selecting the best fit.

| Column | Type | Description |
|---|---|---|
| player_id | string | Unique player identifier |
| player_name | string | Full player name |
| team | string | Player's team abbreviation |
| archetype | string | Assigned archetype label (e.g. "Off-Ball Shooter", "Primary Creator") |
| archetype_score | float | Similarity score to the assigned archetype template (higher = better fit) |
| second_archetype | string | Second-best fitting archetype |
| second_score | float | Similarity score to the second-best archetype |

---

### `player_peer_groups.csv`
Peer group membership for each player. Peer groups are constructed by combining position group and usage tier to ensure role-appropriate comparisons.

| Column | Type | Description |
|---|---|---|
| player_id | string | Unique player identifier |
| player_name | string | Full player name |
| team | string | Player's team abbreviation |
| peer_group_id | string | Unique identifier for the peer group this player belongs to |
| position_group | string | Broad position group (e.g. Guard, Wing, Big) |
| usage_tier | string | Usage tier classification (e.g. High, Medium, Low) |
| peer_group_size | integer | Total number of players in this peer group |

---

### `player_final_evaluations.csv`

| Column | Type | Description |
|---|---|---|
| player_id | string | Unique player identifier |

| shooting_score | float | Shooting dimension score (0–100) |
| playmaking_score | float | Playmaking dimension score (0–100) |
| rebounding_score | float | Rebounding dimension score (0–100) |
| defense_score | float | Defense dimension score (0–100) |
| efficiency_score | float | Efficiency dimension score (0–100) |
| shooting_grade | string | Letter grade for shooting dimension |
| playmaking_grade | string | Letter grade for playmaking dimension |
| rebounding_grade | string | Letter grade for rebounding dimension |
| defense_grade | string | Letter grade for defense dimension |
| efficiency_grade | string | Letter grade for efficiency dimension |

| overall_grade | string | Final letter grade (A through F scale) |
| peer_group_id | string | Peer group the player was evaluated within |

---

## Notes

- Percentile values range from 0 to 100. Higher is better for all metrics except turnover rate and defensive rating (where lower raw values are better — those are inverted before percentile calculation).
- Dimension weights vary by archetype. A Primary Creator's final score weights Playmaking and Efficiency most heavily, while an Interior Forward weights Rebounding and Defense most heavily.
- Raw data files (`data/raw/`, `data/processed/`) are excluded from version control per `.gitignore`. Run the data collection scripts to regenerate them.
